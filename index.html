Le Web 2.0, quelquefois appelé Web participatif, désigne l'ensemble des techniques, des fonctionnalités et des usages qui ont suivi la forme originelle du Web, www ou World Wide Web1, caractérisé par plus de simplicité et d'interactivité. Il concerne en particulier les interfaces et les échanges permettant aux internautes ayant peu de connaissances techniques de s'approprier des fonctionnalités du Web.

Protocole Les protocoles réseau sont un ensemble de règles, de conventions et de structures de données qui dictent la manière dont les appareils échangent des données sur les réseaux. Les protocoles réseau contrôlent tous les aspects de la communication réseau, de l'envoi et de la réception de messages au formatage des fichiers pour différents types de messages. De la même manière que parler la même langue simplifie la communication entre deux personnes, les protocoles réseau permettent aux appareils d'interagir les uns avec les autres grâce à des règles prédéterminées intégrées au logiciel et au matériel des appareils. Un protocole réseau précisera par exemple : Le format des paquets de données. Le système d'adressage. Procédures de vérification des erreurs utilisées.

Une API Web est une interface de programmation d'application pour un serveur Web ou un navigateur Web .
Il s'agit d'un concept de développement Web , généralement limité au côté client d'une application Web (y compris les frameworks Web utilisés), et n'inclut donc généralement pas les détails d'implémentation
du serveur Web ou du navigateur tels que les SAPI ou les API, à moins qu'ils ne soient accessibles publiquement par une télécommande.
